## 项目概览

该项目是一个单页应用 (SPA)，旨在数字化“子弹笔记”方法论。它包含日历视图（年/月/周）、每日记录（Daily Log）、未来日志（Future Log）以及基于 Markdown 的条目编辑功能。
- **核心技术栈**: React, TypeScript, Vite, Tailwind CSS, DaisyUI.
- **主要库**:
    - `date-fns`: 日期处理核心。
    - `react-router-dom`: 路由管理。
    - `@dnd-kit/core`: 拖拽排序功能。
    - `react-markdown`: Markdown 渲染。
    - `framer-motion`: 动画效果。
    - `axios`: HTTP 请求与拦截器。
## 目录结构与模块划分
```text
src/
├── features/             # 核心业务模块
│   ├── auth/             # 认证模块 (登录/注册/Token管理)
│   ├── calendar/         # 日历主视图 (年/月/周视图逻辑)
│   ├── daily/            # 每日详情页 (Daily Log)
│   └── entry/            # 条目组件 (显示/编辑/操作菜单)
├── components/           # 通用组件
│   ├── modals/           # 各种弹窗 (新增/删除/迁移/搜索)
│   └── shared/           # 共享UI (Markdown工具栏/图片上传)
├── hooks/                # 全局 Hooks (主题/多语言/导航)
├── lib/                  # 基础设施 (API拦截器/工具函数)
└── config/               # 静态配置 (颜色/翻译/主题定义)
```
## 核心模块与函数分析
### features/auth
- useAuth.ts
    - `authenticate(username, password, isRegister)`: 处理登录和注册逻辑。登录成功后将 `access_token` 和 `refresh_token` 存入 `localStorage`。
    - `logout()`: 清除本地 Token 并重定向至登录页。
- **`LoginPage.tsx`**: 登录/注册 UI，包含与 `HelpModal` 的交互。
- **`lib/api.ts`**:
    - 配置了 Axios **请求拦截器**：自动将 `localStorage` 中的 Token 注入 Header。
    - 配置了 Axios **响应拦截器**：监听 `401 Unauthorized` 错误，自动调用 `/auth/refresh` 接口刷新 Token 并重试失败的请求（无感刷新）。
### features/calendar
这是应用的主页，也是最复杂的模块，负责时间维度的导航。
- **`useCalendarLogic.ts` (Hook)**:
    - **状态管理**: 管理 `viewMode` (年/月/周) 和 `currentDate` (当前焦点日期)。
    - **数据获取**: `fetchMonthOverview` 负责获取月度概览数据（即日历上的小圆点）。
    - **缓存机制**: 使用 `overviewCache` 状态缓存已加载的月份数据，减少网络请求。
- **`CalendarPage.tsx`**:
    - 作为容器组件，根据 `viewMode` 切换渲染 `YearGrid`（年视图）或 `StandardGrid`（月/周视图）。
    - 处理全局状态（如通过 `location.state` 接收跳转请求）。
- **`StandardGrid.tsx`**:
    - 计算当前视图需要渲染的日期网格。
    - 调用 `CalendarDots` 组件渲染每天的任务状态点（颜色映射逻辑在 `config/colors.ts`）。
### 条目系统 (`features/entry`)
条目（Entry）是子弹笔记的原子单位（任务/灵感/事件）。
- **`EntryItem.tsx`**:
    - **核心控制器**: 管理条目的显示模式（查看 vs 编辑）。
    - **状态流转**: `handleStatusToggle` 函数处理任务状态变更（Open -> Completed）。支持**乐观更新 (Optimistic UI)**，即先更新 UI，再发送 API 请求，提升用户体验。
    - **交互**: 集成了 `MigrateModal`（迁移）、`FutureModal`（移至未来）和 `DeleteModal`。
- **`EntryDisplay.tsx`**:
    - 使用 `getSmartSummary`（来自 `utils/markdownUtils.ts`）清洗 Markdown 内容，生成适合列表展示的摘要。
    - 调用 `MarkdownViewer` 渲染富文本。
- [[EntryEditor]].tsx:
    - 提供文本输入框和 `MarkdownToolbar`。
    - `MarkdownToolbar` 集成了 `ImageUploader`，支持图片上传并插入 Markdown 语法。
### 3.4 每日记录 (`features/daily`)
- **`DailyPage.tsx`**:
    - 展示特定日期的所有条目。
    - **拖拽排序**: 集成 `@dnd-kit`，使用 `SortableContext` 实现条目的拖拽排序。
    - `handleDragEnd`: 计算拖拽后的新顺序，并调用 API 保存。
### 3.5 弹窗系统 (`components/modals`)
使用 HTML `<dialog>` 元素实现，通过 `ref.showModal()` 触发。
- **`AddEntryModal.tsx`**:
    - 支持“每日记录”和“Future Log”两种模式。
    - `handleSubmit`: 根据模式构造不同的 Payload（由 `target_date` 或 `target_month` 区分）发送给后端。
- **`SearchModal.tsx`**:
    - 支持文本搜索和正则搜索。
    - 支持按类型（Task/Idea/Event）筛选。
    - 点击结果通过 `useEntryNavigation` 跳转到对应日期。
- **[[FutureLog|FutureLogModal]].tsx**:
    - 展示未来的任务规划。
    - 支持将条目在不同月份之间拖拽（规划任务）。
## 4. 模块间的交互与合作逻辑
### 4.1 数据流与状态同步
应用没有使用 Redux 等全局状态库，而是通过以下方式协作：
1. **父子组件通信**:
    - `CalendarPage` 将 `handleSilentRefresh` 传递给 `AddEntryModal`。当新条目创建成功后，Modal 调用该回调，触发 `CalendarPage` 重新拉取数据，更新日历上的圆点。
2. **路由与导航**:
    - **`useEntryNavigation.ts`**: 这是一个关键的 Hook。当用户在搜索结果或 Future Log 中点击一个条目时，它判断条目是否有具体日期。
        - 如果有日期：`Maps('/daily/YYYY-MM-DD')`。
        - 如果是 Future Log 条目：跳转回首页并打开 Future Log 弹窗。
    - **`CalendarPage`** 监听 `location.state`，如果发现这就跳转请求，会自动定位到目标日期并高亮。
### 4.2 乐观更新 (Optimistic UI)
为了让应用感觉“极速”，多个组件采用了乐观更新策略：
- **场景**: 用户在 `DailyPage` 点击任务完成。
- **流程**:
    1. `EntryItem` 立即调用 `onOptimisticUpdate` 修改本地状态，UI 瞬间变绿（完成态）。
    2. 后台异步发送 API 请求。
    3. 如果 API 失败，理论上应回滚（当前代码主要侧重于 UI 即时反馈）。
### 4.3 拖拽排序协作
- **`DailyPage`** (列表排序) 和 **`FutureLogModal`** (月度规划) 都使用了 `@dnd-kit`。
- 它们共享了类似的逻辑：监听 `onDragEnd` -> 计算数组新顺序 (`arrayMove`) -> 更新本地 State -> 静默发送 API 请求保存顺序。
### 4.4 国际化与主题
- **`useTranslation.tsx`** 和 **`useTheme.ts`** 通过 Context API 向整个应用树提供语言包（`config/translations.ts`）和深色模式状态。
- 所有组件（如 `CalendarHeader`）都通过 `useTranslation` 获取文本，确保切换语言时无缝更新。
## 5. 总结
这份代码构建了一个结构清晰、交互细腻的 React 应用。
- **优点**: 模块化程度高，大量使用了复用组件（如 `EntryItem`, `MarkdownViewer`），并且在 UX 上做了很多优化（乐观更新、拖拽排序、Markdown 实时预览）。
- **核心逻辑**: 围绕 `Calendar`（时间轴）和 `Entry`（内容）两个核心实体展开，通过 `Modal` 进行内容的创建和调度。
