#### 1. 组件概述
`CalendarPage` 是应用的核心主页，承载了**日历视图**、**任务列表**、**拖拽排序**以及**数据管理**的核心逻辑。它采用“响应式 + 乐观更新”的设计哲学，确保用户操作（如打钩、删除、新建）能得到即时反馈。
#### 2. 核心状态管理 (State Architecture)
- **视图模式 (`viewMode`)**: 支持 `year` (年度热力图), `month` (标准月历), `week` (周视图)。
- **日期游标**:
    - `currentDate`: 决定日历面板显示哪个月份/年份（导航用）。
    - `selectedDate`: 决定下方列表显示哪一天的任务（选中用）。
- **双层缓存 (`Cache`)**:
    - `overviewCache`: 存储轻量级的日概览信息（仅包含 ID、类型、状态），用于渲染日历上的圆点。
    - `dailyCache`: 存储完整的任务详情数组，用于渲染下方列表。这种分离设计极大减少了不必要的网络请求。

#### 3. 关键技术点

- **乐观更新 (Optimistic UI)**:
    - 所有增删改查操作（CRUD）都先修改本地 `Cache`，立即渲染 UI，然后再发送网络请求。
    - **联动更新**: 修改列表数据时，会自动同步重新计算日历上的圆点 (`regenerateDotsFromList`)，保证视图一致性。
- **拖拽排序 (DnD)**:
    - 集成 `@dnd-kit`，支持长按拖拽排序。
    - 实现了复杂的**列表乐观排序**：拖拽结束瞬间交换数组位置，若后端 API 失败则回滚。
- **手势操作**:
    - 使用 `TouchSensor` 监听滑动，支持左右滑动切换月份，上下滑动切换周/月视图。
- **动画系统**:
    - 使用 `Framer Motion` 处理视图切换（年/月切换）的过渡动画，以及列表加载的淡入效果。
